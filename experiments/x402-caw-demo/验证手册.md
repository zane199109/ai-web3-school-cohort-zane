# x402-CAW Demo 验证手册

## 前置条件

- 终端在 WSL 环境
- Foundry (forge, cast, anvil) 已安装
- Go 1.22+ 已安装（`/usr/local/go/bin/`）
- 网络代理已处理（`unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY ALL_PROXY no_proxy NO_PROXY`）

## 2. HTTP 402 联动三场景验证

模拟真实 Agent 通过 HTTP 402 协议与 Provider 交互，CAW 在链下预检后决定是否执行链上支付。

### 2.1 启动 Provider（终端 A，长期运行）

Provider 即是 Mock API Server —— 模拟一个带 HTTP 402 Paywall 的 AI Inference 服务，提供三个场景路径。

```bash
cd ~/ai-web3-school-cohort-zane/experiments/x402-caw-demo/backend-go
go run ./provider/main.go
```

预期输出：
```

  http://127.0.0.1:8081/api/success      → 合法请求 — 10 USDC 给白名单地址
  http://127.0.0.1:8081/api/phishing     → 钓鱼攻击 — 10 USDC 给未授权地址
  http://127.0.0.1:8081/api/overlimit    → 超额请求 — 200 USDC 超出日预算
```

> 保持此终端运行。

## 常见问题

**Q: `cast send` / `forge create` 报 `HTTP error 502`**
A: WSL 代理劫持了本地请求。运行前确保 `unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY ALL_PROXY no_proxy NO_PROXY`

**Q: `go: command not found`**
A: Go 在 `/usr/local/go/bin/go`，运行前执行 `export PATH="/usr/local/go/bin:$PATH"`

**Q: Anvil 掉线了**
A: 重新执行 `bash script/setup_and_deploy.sh`（会 pkill 旧 anvil 并重新启动+部署）

**Q: `address already in use` 端口 8081 或 8545**
A: `fuser -k 8081/tcp` 或 `fuser -k 8545/tcp` 杀掉旧进程再启动
