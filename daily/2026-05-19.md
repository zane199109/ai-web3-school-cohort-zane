### 2.5 国内受限网络环境的 GitHub 操作经验

在本次配置中验证了受限网络下 GitHub 的最佳实践：
1. HTTPS 到 GitHub（443端口）超时，必须使用 SSH（22端口）或 PAT 直接认证。
2. `gh auth login` 交互流程在 WSL pty 背景进程中容易卡住，设备码过期后需重新生成。
3. 最可靠的方案：创建 fine-grained PAT（需开启 Administration 写入权限），用 `--with-token` 模式完成认证。
4. git 协议设为 SSH 后可正常 clone、push、pull。

### 3.2 踩坑总结

1. `gh auth login -p ssh` 在 WSL pty 背景进程中交互困难——设备码不断超时，用户浏览器授权后进程未正确检测到授权状态。
2. **教训：在受限网络 + WSL 环境下，优先使用 `gh auth login --with-token` + fine-grained PAT 方案。**
3. fine-grained PAT 创建仓库需要开启 Account permissions → Repository creation（Write），而不是仓库级权限。
4. Git 首次 commit 需要设置 user.name 和 user.email，否则会报 "Author identity unknown"。
5. 空目录不会被 git 跟踪，需要用 `.gitkeep` 占位文件才能推送到 GitHub。
