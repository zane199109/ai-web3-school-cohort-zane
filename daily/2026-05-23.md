## 二、实操核心踩坑 & 问题复盘（核心困惑）

本次借助智能学习 Agent 辅助作业推进过程中，暴露了大量 AI 落地实操的真实问题，也是目前 AI Agent 普遍存在的能力短板，同时也让我区分清了「**使用方式问题**」和「**模型本身能力问题**」。

**1\. 严重的幻觉与虚假汇报问题**

Agent 经常自主编造虚假内容、虚构未完成的步骤，甚至在任务并未落地、代码未跑通、内容未整理的情况下，直接汇报「任务已全部完成」，无法真实感知实际执行状态，属于典型大模型幻觉缺陷。

**2\. 无视格式约束，输出极不稳定**

即便多次提前指定严格的输出格式、分段要求、排版规范、术语结构，Agent 依旧频繁乱输出、不按模板、不分段、冗余加粗、结构混乱，每次生成结果风格不统一，一致性极差。

**3\. 答非所问，理解指令能力弱**

面对稍微复杂的课程任务、多条件约束、分层要求，Agent 无法精准解析核心需求，容易跑偏、遗漏关键要点、回避核心问题，只会生成通用套话，无法针对性落地专属任务。

**4\. 极度依赖人工反复调试**

几乎无法一次生成合格内容，需要反复修改 Prompt、补充约束、纠错调参，多次迭代后才能输出符合作业标准的结果，不仅没有提升效率，反而增加了调试成本。

## 三、核心困惑总结

目前无法界定问题根源：到底是 **自己使用 Agent 的方式不对、Prompt 工程不到位**，还是 **当前基础大模型本身的 Agent 能力上限不足**，导致任务理解、格式遵守、真实执行校验能力薄弱。
